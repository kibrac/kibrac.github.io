重装系统备忘录

February 27, 2026 1 min read

Git 配置

以下代码均在 Git Bash 中运行。

设置用户信息

1
2

git config --global user.name "Name Here" //设置用户名
git config --global user.email admin@example.com //设置自己的邮箱

Git 通过 SSH 连接到 Github

本地创建密钥文件

1

ssh-keygen -t rsa -C "这里换上你的邮箱"

执行命令后一路点击确认即可。

其中涉及到的操作包含确认 Key 的保存路径，如果路径下原来已有密钥文件则确认是否覆盖，创建密码——直接回车代表不需要密码，确认密码。）

命令执行完成后会在上述提到的保存路径中生成两个文件，分别名为 id_rsa 和 id_rsa.pub。

Github 配置

在 Github 的 Settings 左侧边栏中选择 SSH and GPG keys，点击 New SSH key，Title 可以自己取，Key type 保持为 Authentication Key，然后将之前生成的 id_rsa.pub 用文本工具例如记事本打开，将其中的内容粘贴到输入框中保存即可。

使用下列命令可以测试是否设置成功。

1

ssh -T git@github.com

首次配置需要确认一次，输入 yes 之后出现欢迎语句即表明配置成功，如下图示例。

PicGO 配置（以 Github 图床为例）

生成 Github Token

  1. 在 GitHub 的 Settings 左侧边栏中选择 Developer settings；
  2. 点击 Personal access tokens 选择 Tokens(classic) 生成一个新 Token 并确保勾选了 repo 权限；
  3. 复制生成的 Token（注意只会显示一次，所以此时应当妥善保存）。

PicGo 配置

  1. 在软件的图床设置中选择 Github；
  2. 点击加号新建一个配置，从上到下依次是图床配置名（可自定义）、设定仓库名（Github中对应仓库的名字）、设定分支名、设定 Token（即此前生成的 Token）、设定存储路径（如可指定仓库下的特定文件夹）、设定自定义域名（如果有的话，否则可置空）
  3. 保存配置。
